WHY・為什麼要這樣做

作者看不見自己假設裡的錯

人如此，AI 也如此：根植於作者自身假設的缺陷，對作者是「結構性看不見」的。所以同一個 AI 自己檢查自己 ≈ 沒檢查——它會帶著同一套盲點再讀一次，然後說「沒問題」。換一家公司訓練出來的模型，盲點分佈不同，你的漏洞恰好是它的直覺。

實測案例：Claude 把明顯的洞都修完後，仍殘留一條「權限失效時繼續顯示孩子隱私資料」的路徑——是審查方在第 3 輪逼出來的。作者修得掉明顯的錯，修不掉自己想不到的錯。

CALL・怎麼呼叫

九成情境：你什麼都不用做

主要路徑・全自動 寫計劃 → 自動被攔下審查

只要 AI 寫了「會被照著執行的文件」（計劃、規格、或掛了標記的 skill 說明），工作回合結束的瞬間，守門程式就攔下它：沒有審查證明，這回合結束不了。審查不靠記得叫，靠結構發生。

臨時點名 一句話：「叫 codex 審這份文件」

或斜線指令 /cross-model-review 檔案路徑，任何檔案隨時送審。

反向互審 「幫我審這份 GPT 寫的東西」

角色對調，Claude 當審查方，用同一套找碴協定。唯一的規則：作者不能是自己——自審偽裝成互審會被協定擋下。

RUN・怎麼執行

一次審查的完整循環

攔截守門程式

回合結束時掃描：這回合寫過計劃文件嗎？檔尾有蓋「防偽章」嗎？沒有——擋下，強制進入審查。

證偽式審查審查方

審查方收到的指令不是「看看有沒有問題」，而是「把這份文件當成待推翻的假說」：親自打開真實的程式碼與檔案，逐條查證文件的每個宣稱——文件說某模組存在？去確認。文件承諾某功能？找出是哪個步驟交付的。每輪最多報 5 條最重要的問題，不淹沒對方。

逐輪辯論起草方審查方

起草方對每條意見二選一：修（改文件）或反駁（給具體理由）。審查方對每條反駁必須明確表態：讓步或堅持。同一位審查者記得每一輪說過的話——不准敷衍蓋章、不准為了收工假裝被說服，每筆爭點都記入落盤的帳本。

共識，或交給人兩個出口

審查方說 APPROVED → 過關。吵滿 5 輪還沒共識 → 不打消耗戰，自動整理「雙方立場對照表」交給人類裁決。人永遠是最後防線。

蓋防偽章守門程式

文件尾端蓋上審查章：時間、輪數、結論、審查者、內容指紋。審完之後哪怕偷改一個字，指紋立刻對不上，章自動失效、重新被攔。光有章還不夠——完整的審查證據（帳本、每輪原文）必須同時歸檔，缺一不放行。

EVOLVE・怎麼自我進步

每次失誤都變成一道新關卡

  • 事故驅動進化——每發現一次漏抓或誤報，就新增一條自動檢查＋一條迴歸測試，錯誤不二犯。上線第一天它就抓到自己兩個誤報，當天修掉，測試從 20 條長到 31 條。
  • 審查帳本全落盤——每次審查的爭點、誰讓步、幾輪收斂，全部存檔可回顧：哪類問題反覆出現、審查方哪類意見常被推翻，都有據可查。
  • 稽核稽核者——一份埋了 3 個已知地雷的假計劃當「考卷」，定期丟給審查方重考（例如它換版本後），驗證它還抓得到全部地雷。目前基線：3/3 全中。

GAIN・好處

你得到什麼

品質

盲點互補

不同公司訓練的模型，系統性盲點不重疊。實測：埋 3 個缺陷的計劃，審查方第 1 輪全數抓出，還加碼揪出 6 個沒人埋的真問題。

成本

在最便宜的階段抓錯

審的是計劃而不是成品——錯誤在被照著執行、繁殖成一堆程式碼之前就被攔下，修正成本差一個數量級。

紀律

制度強制，不靠自覺

守門的是確定性程式，不是 AI 的「我記得要審」。忘記、偷懶、想跳過，結構上都行不通。

誠信

雙向防敷衍

審查方被禁止湊數拖戲，起草方被禁止照單全收與假投降——爭點只能靠論證或證據收斂，兩邊都有防腐條款。

可稽核

證據鏈完整

每次審查留下帳本、逐輪原文、裁決紀錄與內容指紋——事後隨時可驗「這份文件是真的被審過，審的就是這個版本」。

主權

人是最後防線

AI 僵持不下時交人裁決；高風險文件即使 AI 全數通過，仍要人類簽核才算完成。

幫我把 github.com/med95Albert/cross-model-review 這個技能裝到 ~/.claude/skills/cross-model-review，然後執行裡面的 install.sh 完成 Stop hook 註冊

裝完重開一個 Claude Code session 即生效。偏好原生外掛的話：/plugin marketplace add med95Albert/cross-model-review → /plugin install cross-model-review@cross-model-review。前置需求：Claude Code＋Python 3＋（建議）已登入的 OpenAI Codex CLI。原始碼與完整說明在 GitHub。

誠實的三個限制

  • 防無心，不防蓄意——所有證明終究是 AI 寫的，防偽設計把「抄捷徑」堵死，但擋不住刻意造假；所以高風險輸出保留人類簽核。
  • 要養兩個訂閱——跨模型的前提是兩家的 CLI 都裝好登入（沒有審查方時會降級成「同模型、全新視角」的次佳方案）。
  • 審查有成本——每次多輪高強度推理，一次約數萬 token。所以只審「會被照著執行的文件」，草稿筆記不納管。
