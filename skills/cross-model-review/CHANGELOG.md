# cross-model-review 版本史

- **v1.1.1 (2026-07-09)**: 校準模型綁定。使用者換裁判模型（gpt-5.5 → gpt-5.6-sol）時發現的洞：calibration 只綁 codex CLI 版本，純換 config.toml 的 model 不會失效——但換模型＝換裁判。修法：calibrate.sh 記錄 `model`（讀 config.toml，`CROSS_REVIEW_JUDGE_MODEL` 可覆蓋）；`--calibration-check` 比對現行模型，不符或紀錄缺欄（舊格式）即 invalid；迴歸 M5c/M5d。本次順帶：codex CLI 0.142.3→0.144.1（0.142 不認得 5.6-sol，會以 fallback metadata 劣化執行）。教訓：**裁判的識別＝CLI 版本×模型**，缺一不可。

- **v1.1 (2026-07-08)**: 裁判校準層（與同行交流「跨模型互審方法論」後的移植）。①**黃金集校準＋fail-closed**：`calibrate.sh`＋`gold-seed/`（4 雷題各測一種能力＋2 乾淨題測假陽性方向——之前只測「會不會漏抓」從沒測「會不會亂抓」）→ `calibration.json`（效期 30 天、codex 換版即失效）；probe 回報 POSTURE，RED＝所有工件視同 🔴（APPROVED 後仍需人工簽核）。②**仲裁→黃金集飛輪**：Step 7 仲裁與被人推翻的簽核歸檔 `gold-candidates/`（僅「人類獨立定讞」入集；「人同意 AI」不算獨立訊號——同行的獨立錨點紀律）。③**防作弊**：Gate 4 新增 D9（ledger 稱 CONCEDE 必須在 r*.txt 找得到原文，防代筆）＋D10（校準狀態，WARN=升級簽核）；held-out 與逐題留檔防校準過擬合。④誠實界線 #7（跨模型可能一起錯——同行實測）＋#8（校準是姿態級非逐審級；hook 不驗校準）。⑤RESOLVED-CAPITULATED 明文不得作為審查有效之證據。刻意不搬：一次性 JSON 判決（對文件審查是退步）、自動優化迴圈（風險集中處）。
  ⑥dogfood v1.1（codex 三輪 APPROVED）：R1 五條全修——RED 姿態機械化（signoff.txt＋D10 三態＋D11）、--state-root 統一、仲裁三處一致＋D8 升 FAIL、D9 改計數下界、校準 runs/ 逐題落檔＋history；R2 三條全修——證據綁版（meta.sha＝marker.sha，hook 與 D12 雙驗，攻擊迴歸 C4）、D11 tier 必填不 fail-open、wordcount 夾具改 OSError 總類。基線三跑：FAIL（乾淨題不自足，0/2）→ PASS（5/6 容忍已知瑕疵，被 codex 抓）→ **PASS（6/6 乾淨）**。合成測試 52 案例全綠。黃金集首批教訓：乾淨題須「空倉庫自足」；容忍額度是給裁判偶發挑剔的，不是給已知壞題的；審查者會親讀活體狀態檔——它驗的是現實不是說法。

- **v1.0 (2026-07-04)**: 初版。合併 Gary Chen codex-review kit（機制：Stop hook 強制、證偽式審查 prompt、逐輪 CONCEDE/MAINTAIN 共識、resume by thread_id、每審隔離、exit code 鐵律）與 tier-s v2.1 治理（風險分級、5 Gates、人類最後防線、SkillOpt Edit Budget）。
  相對 kit 的六個新設計：①可插拔審查者（codex 優先、fresh subagent 備援，rounds 2+ 用 SendMessage 對應 resume）②sha 綁定 marker（審後改動自動失效；`review-gate.py --sha/--check` 與 hook 單一實作）③輪數上限 5＋人類仲裁出口（拆掉「唯一出口是 APPROVED」的假投降壓力）④Gate 2 帳本落盤（ledger.md 供 Gate 4 機器驗，防 MAINTAIN 被悄悄跳過）⑤審查端 Edit Budget=5（防 issue 洪水拖戲）⑥SKILL.md 監看採 sentinel opt-in（`cross-model-gated`，防監看過寬→審查疲勞，這是預判的第一事故並前置緩解）。
  設計時已知的坑（原 kit 明防，保留）：stale read、`--last` 跨審錯接、共用暫存路徑。
  同日兩輪對抗修正：①上線首個 stop 即抓到 sentinel 裸子字串誤納管（tier-s 教學句中招）→ 改「獨立一行才算 opt-in」＋迴歸測試 J。②Step 9.5 opus audit 回報 5 P0 → 接受 4＋1 P1：round-1 heredoc 改 unquoted（quoted 版把字面佔位符送審查者）、stop_hook_active 由靜默放行改「exit 1＋stderr 可見警告」、放行改 marker＋`_state` 證據雙要件（堵自簽 marker 一行過關）、transcript 全檔掃描（原 200 則視窗長回合漏抓）、D5 路徑正則修截斷假 WARN；拒絕 1（豁免自身目錄——自我納管是刻意設計），理由見 rejected-edits.md。③codex dogfood（2 輪 APPROVED）：R1 提 4 issue → 3 修（hook 證據升四要件含 gate4 FAIL=0、誠實界線#6 監看範圍聲明、Step 0 自足 fallback）＋1 反駁成立（thread_id 事件流機制，codex CONCEDE）。④Gate 4 D8 修正：改解析真 marker 的 verdict 欄，不再裸 grep 全文（與 sentinel 事故同類，迴歸測試 H2）。⑤使用者實機回報：D4 訊息 `$LR` 緊貼全形括號在 bash 3.2＋zh_TW.UTF-8 下解析成未定義變數名而中止——改 `${VAR}` 並全腳本掃描無第二處；驗收改以使用者條件（/bin/bash＋zh_TW.UTF-8）執行。⑥品質實測（埋 3 雷 plan，codex r1 全抓＋加碼 6 真問題、4 輪 APPROVED＋人類簽核）暴露 D5 第二型假陽性：Next.js 動態路由片段 `[childId]/page.tsx` 被擷成 `/page.ts`——D5 擷取改用 python regex（左界排除 `]`、`)`、`}`；右界防 `.ts` 吃 `.tsx`；副檔名清單加 tsx/jsx），迴歸測試 G3×3。⑦新增品質回歸工具 `_state/cross-model-review/quality-test.sh`＋無劇透夾具（3 雷；夾具內嚴禁提示性註解——首版註解劇透被 codex 讀到，已除）；乾淨基線 3/3 PASS。測試組 31 案例全綠。
